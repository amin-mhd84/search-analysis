## 📌 معرفی پروژه

این پروژه بیش از ۶۶ هزار رکورد جست‌وجوی کاربران در پلتفرم رزرو سفر **مستربلیط** را تحلیل می‌کند تا به پرسش‌های کلیدی کسب‌وکاری پاسخ دهد:

- کدام سرویس‌های سفر (اتوبوس، پرواز، قطار، هتل، تاکسی، کشتی) محبوب‌ترند؟
- کاربران بیشتر به کدام شهرها و استان‌ها سفر می‌کنند؟
- کدام شهرهای پرجمعیت هنوز به‌اندازه‌ی کافی از پلتفرم استفاده نمی‌کنند؟ (فرصت بازار)

**چرا این پروژه؟**
- 🧹 پیش‌پردازش متنی واقعی روی داده‌ی خام و ناتمیز (نام شهرها به همراه جزئیات پایانه/فرودگاه)
- 🗺️ ادغام دو منبع داده (جست‌وجوهای کاربر + دیتاست جغرافیایی شهرهای ایران) برای تحلیل عمیق‌تر
- 📊 نمودارهای تعاملی با Plotly، از جمله نمایش جغرافیایی توزیع تقاضا
- 💡 هر یافته با یک پیشنهاد کسب‌وکاری مشخص همراه است، نه فقط عدد خام

## 📊 دیتاست

| فایل | توضیح |
|---|---|
| `mrbilit_search.json` | ۶۶٬۱۹۵ رکورد جست‌وجوی کاربر با ستون‌های `ServiceType` (نوع سرویس)، `TypedStrings` (رشته‌های تایپ‌شده به ترتیب زمانی) و `AcceptString` (رشته‌ی نهایی انتخاب‌شده) |
| `iran_cities.csv` | اطلاعات جغرافیایی و جمعیتی بیش از ۱۲۰۰ شهر ایران — نام فارسی/انگلیسی، استان، مختصات، و جمعیت سرشماری ۲۰۱۱ و ۲۰۱۶ (برگرفته از ویکی‌پدیا) |

## 🔬 تحلیل گام‌به‌گام

### ۱. تحلیل اکتشافی داده (EDA)

پیش از پاسخ به سؤالات اصلی، نگاهی کلی به داده می‌اندازیم: بررسی shape، مقادیر گمشده، و به‌خصوص مقادیر یکتای ستون `ServiceType`. این گام مهم است چون در نسخه‌ی اولیه‌ی این تحلیل، فرض غلطی روی تعداد ثابت سرویس‌ها (فقط ۵ سرویس شناخته‌شده + یک دسته‌ی پیش‌فرض) باعث ایجاد یک باگ خاموش می‌شد که هیچ خطایی هم نمی‌داد.

```python
print('shape:', data.shape)        # (66195, 3)
data.isnull().sum()                # بدون مقدار گمشده
data['ServiceType'].unique()       # ['flight' 'hotel' 'bus' 'taxi' 'train' 'ship']
```

نتیجه: داده کامل و بدون مقدار گمشده است و دقیقاً ۶ نوع سرویس دارد — همان چیزی که در ادامه‌ی تحلیل روی آن حساب می‌کنیم.

### ۲. محبوبیت سرویس‌ها

به ازای هر سرویس، درصد آن از کل جست‌وجوها را با `value_counts(normalize=True)` محاسبه می‌کنیم — به‌جای شمارش دستی با حلقه‌ی `if/elif` که هم کندتر است و هم در برابر مقادیر پیش‌بینی‌نشده شکننده.

**یافته:** اتوبوس با اختلاف زیاد محبوب‌ترین سرویس است (۴۰.۲٪ از جست‌وجوها)، و پس از آن پرواز (۲۵٪) و قطار (۱۷.۹٪) قرار دارند. هتل ۱۰٪ و تاکسی ۵.۲٪ را به خود اختصاص داده‌اند. کشتی با ۱.۷٪ کمترین سهم را دارد که با توجه به محدود بودن مسیرهای دریایی در ایران، منطقی به نظر می‌رسد.

### ۳. پیش‌پردازش نام شهرها

برخی مقادیر ستون `AcceptString` به‌شکل «نام شهر - جزئیات» نوشته شده‌اند (مثلاً «شیراز - پایانه امیرکبیر» یا «تهران - فرودگاه امام خمینی»). این‌ها را با یک تابع ساده به نام خام شهر تبدیل می‌کنیم تا در ادامه بتوانیم بر اساس شهر گروه‌بندی کنیم:

```python
def extract_city(value: str) -> str:
    """Keeps only the city part of an AcceptString value (before ' - ' if present)."""
    return value.split('-')[0].strip() if '-' in value else value.strip()
```

### ۷. شهرهای پرجمعیت اما کم‌جست‌وجو

## 🐛 رفع باگ نسبت به نسخه‌ی اولیه

در نسخه‌ی اولیه‌ی این تحلیل، بخش «شهرهای پرجمعیت اما کم‌جست‌وجو» به‌اشتباه شهرها را با لیست **استان‌های** پرجست‌وجو (`top_provinces`) مقایسه می‌کرد، در حالی که طبق صورت مسئله باید با لیست **شهرهای** پرجست‌وجو (`top_cities`) مقایسه می‌شدند:

```python
# ❌ نسخه‌ی قبلی (باگ‌دار)
for i in census_500['City FA']:
    if i not in top_provinces:   # مقایسه‌ی نام شهر با نام استان!
        not_in_top.append(i)

# ✅ نسخه‌ی اصلاح‌شده
not_in_top = [city for city in census_500['City FA'] if city not in top_cities]
```

این تناقض نوع داده باعث می‌شد عملاً همه‌ی شهرهای پرجمعیت به‌عنوان «کم‌جست‌وجو» شناسایی شوند، چون نام هیچ شهری با نام هیچ استانی برابر نیست. در این نسخه این باگ رفع شده و نتیجه اکنون معنادار است (فقط ۳ شهر از ۱۸ شهر پرجمعیت، واقعاً کم‌جست‌وجو هستند).
